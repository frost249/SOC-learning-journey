## Practical Investigations

### 1. Shadow Trace & Threat Detection (Linux, Windows, Web, Network)
> Tools: PE Bear, Windows Event Viewer, Sysmon, Audit Logs, Web Server Logs

**Scenario:** Analyzed a suspicious file to uncover hidden clues and trace the source of infection, alongside platform-specific threat detection across Linux, Windows, Web, and Network environments.

**Shadow Trace — Malware File Analysis**

- Analyzed malicious files and extracted Indicators of Compromise (IOCs)
- Used **PE Bear** to perform static analysis on a suspicious `.exe` file:
  - Extracted file hashes (MD5/SHA256)
  - Identified binary architecture
  - Found loaded libraries — specifically one related to **socket communication**, indicating potential C2 or data exfiltration capability
- Investigated **Living Off The Land (LOLBAS)** attack techniques — abuse of legitimate Windows binaries to evade detection

**Linux Threat Detection**

- Analyzed **audit logs** for malicious command execution patterns
- Reviewed **auth logs** to detect brute force login attempts against SSH
- Investigated **web server logs** to identify malicious HTTP requests and attacker behavior

**Windows Threat Detection**

- Used **Windows Event Viewer** to reconstruct the sequence of malicious events
- Detected brute force attempts via authentication event correlation
- Used **Sysmon** for deep process-level visibility — analyzed process creation, network connections, and file writes for malicious indicators

| File Hash | Extracted via PE Bear static analysis |
| Library | Socket communication DLL loaded by malicious binary |
| Technique | SSH Brute Force |
| Technique | Malicious process execution chain |

### 2. SIEM Investigation — ItsyBitsy & Benign (TryHackMe)
> Tools: Splunk, Elastic Stack (ELK), LOLBAS Reference

**Scenario:** An IDS alert flagged suspicious process execution on an HR department host. The objective was to triage alerts and identify attacker behavior using SIEM tools.

**Investigation Steps:**

**Step 1 — SIEM Log Analysis**
- Used Splunk SPL queries to filter suspicious process activity:
```spl
index=* sourcetype=* process*
index=* certutil
```
- Used Elastic Stack to correlate log events across sources

**Step 2 — User Impersonation Detected**
- Identified a duplicate account impersonating user `Amelia`
- Confirmed via authentication log correlation

**Step 3 — LOLBAS Technique (certutil.exe)**
- Traced `certutil.exe` execution by user `Haroon` via PowerShell
- Cross-referenced with [GitHub LOLBAS](https://lolbas-project.github.io/) — confirmed abuse for payload download
- `certutil.exe` used to bypass security controls and download malicious payload

**Key Findings:**
- Suspicious process execution flagged by IDS
- Credential impersonation via duplicate user account
- LOLBAS abuse: `certutil.exe` used for payload delivery

**MITRE ATT&CK Mapping:**

| Technique ID | Name |
|---|---|
| T1036 | Masquerading |
| T1218.003 | certutil (LOLBAS) |
| T1046 | Network Service Discovery |
