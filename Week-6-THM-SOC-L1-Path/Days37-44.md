## Practical Investigations

### 1. Attack Chain Investigation
> Tools: Sysmon Viewer, Wireshark, Windows Event Viewer, PowerShell

**Scenario:** A user downloaded a malicious `.doc` file via `chrome.exe`. The objective was to trace the full attack chain from initial infection to persistence.

**Investigation Steps:**

**Step 1 — Initial Infection**
- Used Sysmon Viewer to trace the malicious `.doc` download via `chrome.exe`
- Identified origin domain: `phishteam.xyz` *(lab environment only)*

**Step 2 — Payload Execution**
- Located suspicious Process ID in Windows Event Viewer (Sysmon logs)
- Found a Base64-encoded PowerShell command — decoded it to reveal a startup persistence mechanism
- Command downloaded secondary payload: `first.exe`

**Step 3 — C2 Communication**
- Sysmon Viewer revealed C2 domain: `resolvecyber.xyz` *(lab environment only)*

**Step 4 — Network Traffic Analysis**
- Wireshark analysis confirmed:
  - Credential theft activity
  - Reverse SOCKS proxy communication
  - Download of additional payloads

**Step 5 — Persistence & Privilege Escalation**
- Attacker executed `spf.exe` and `final.exe` to maintain C2 and escalate privileges
- Gained administrative access
- Created persistence and added 2 new user accounts to the system

**IOCs Identified:**

| Type | Indicator |
|---|---|
| Domain | phishteam.xyz |
| Domain | resolvecyber.xyz |
| File | first.exe |
| File | spf.exe |
| File | final.exe |
| Technique | Base64-encoded PowerShell |
| Technique | Reverse SOCKS proxy |

**MITRE ATT&CK Mapping:**

| Technique ID | Name |
|---|---|
| T1059.001 | PowerShell |
| T1105 | Ingress Tool Transfer |
| T1071 | Application Layer Protocol (C2) |
| T1547 | Boot/Logon Autostart Persistence |
| T1136 | Create Account |
| T1090 | Proxy (SOCKS) |

### 2. SIEM Investigation — ItsyBitsy & Benign (TryHackMe)
> Tools: Splunk, Elastic Stack (ELK), LOLBAS Reference

**Scenario:** An IDS alert flagged suspicious process execution on an HR department host. The objective was to triage alerts and identify attacker behavior using SIEM tools.

**Investigation Steps:**

**Step 1 — SIEM Log Analysis**
- Used Splunk SPL queries to filter suspicious process activity:
```spl
index=* sourcetype=* process*
index=* certutil
```
- Used Elastic Stack to correlate log events across sources

**Step 2 — User Impersonation Detected**
- Identified a duplicate account impersonating user `Amelia`
- Confirmed via authentication log correlation

**Step 3 — LOLBAS Technique (certutil.exe)**
- Traced `certutil.exe` execution by user `Haroon` via PowerShell
- Cross-referenced with [GitHub LOLBAS](https://lolbas-project.github.io/) — confirmed abuse for payload download
- `certutil.exe` used to bypass security controls and download malicious payload

**Key Findings:**
- Suspicious process execution flagged by IDS
- Credential impersonation via duplicate user account
- LOLBAS abuse: `certutil.exe` used for payload delivery

**MITRE ATT&CK Mapping:**

| Technique ID | Name |
|---|---|
| T1036 | Masquerading |
| T1218.003 | certutil (LOLBAS) |
| T1046 | Network Service Discovery |
